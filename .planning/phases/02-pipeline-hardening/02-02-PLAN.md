In `grantflow/app.py`, modify the `lifespan` async context manager:

1. Import at top of file:
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger
import asyncio
from grantflow.ingest.run_all import run_all_ingestion
```

2. Inside `lifespan`, after `init_db()`:
```python
scheduler = AsyncIOScheduler()
scheduler.add_job(
    lambda: asyncio.get_event_loop().run_in_executor(None, run_all_ingestion),
    CronTrigger(hour=2, minute=0, timezone="UTC"),
    id="daily_ingestion",
    replace_existing=True,
    misfire_grace_time=3600,  # tolerate up to 1h delay (e.g. server restart at 02:00)
)
scheduler.start()
logger_app.info("APScheduler started — daily ingestion at 02:00 UTC")
yield
scheduler.shutdown(wait=False)
```

Note: `run_all_ingestion` is synchronous (uses sync SQLAlchemy sessions). Run it in an executor so it doesn't block the event loop. Use `misfire_grace_time=3600` so a restart that causes a missed 02:00 job still runs if the server comes back up within an hour.

The scheduler does NOT need a persistent job store for this use case — in-memory is fine since the job is registered from code on every startup.

Do NOT add a manual trigger endpoint yet — keep scope to automated scheduling only.
