```python
SEARCH2_ENDPOINT = f"{GRANTS_GOV_REST_API_BASE}/search2"
REST_PAGE_SIZE = 25        # API maximum per page
MIN_REST_THRESHOLD = 100   # fewer records than this = treat REST as unreliable
MAX_REST_PAGES = 200       # 200 × 25 = 5,000 record cap; prevent infinite pagination
```

Function `_ingest_via_rest(session) -> dict | None`:
- Returns a stats dict on success, or `None` if REST is unavailable/unreliable (caller falls back to XML)
- Paginates through `search2` fetching "posted" and "forecasted" opportunities
- For each page:
  - POST to `SEARCH2_ENDPOINT` with JSON body (rows=REST_PAGE_SIZE, startRecordNum=page*REST_PAGE_SIZE)
  - On any 5xx or connection error: log warning, return None (trigger XML fallback)
  - On 200 with `totalOpportunityCount == 0` or len(oppHits) == 0: stop pagination
  - Map each oppHit to an Opportunity dict using the field mapping in the interface above
  - Dates: use `_normalize_date()` (already exists, handles multiple formats)
  - CFDA numbers: `", ".join(opp.get("cfdaList", []))` — REST returns a list
  - `id = f"grants_gov_{opp['id']}"` — same composite key format as XML path
  - `source = "grants_gov"` — same source tag, so records from REST and XML are deduplicated by id
- After pagination, if `records_processed < MIN_REST_THRESHOLD`: log warning and return None
- Otherwise: upsert all records (using the existing `_upsert_batch()` helper), return stats dict with `extra={"path": "rest", "pages_fetched": page_count}`

Modify `ingest_grants_gov()` to implement the REST-first, XML-fallback strategy:

```python
def ingest_grants_gov() -> dict:
    # ... existing setup (IngestionLog, session creation) ...

    rest_stats = None
    xml_stats = None

    if GRANTS_GOV_USE_REST:
        # Force REST-only mode (config flag for testing)
        logger.info("mode=rest_only (GRANTS_GOV_USE_REST=true)")
        rest_stats = _ingest_via_rest(session)
        if rest_stats is None:
            stats["status"] = "error"
            stats["error"] = "REST API unavailable and REST-only mode is set"
            return stats
    else:
        # Try REST first
        logger.info("mode=rest_first attempting REST API")
        rest_stats = _ingest_via_rest(session)

        if rest_stats is not None:
            logger.info(
                "rest_succeeded",
                records=rest_stats["records_processed"],
                path="rest"
            )
            # Merge REST stats into main stats
            stats.update(rest_stats)
            stats["extra"] = json.dumps({"path": "rest"})
        else:
            # REST unavailable or returned too few records — fall back to XML
            logger.info("rest_fallback", reason="REST returned None, using XML extract")
            xml_stats = _ingest_via_xml(session)  # existing XML logic extracted into helper
            if xml_stats is not None:
                stats.update(xml_stats)
                stats["extra"] = json.dumps({"path": "xml"})
            else:
                stats["status"] = "error"
                stats["error"] = "Both REST and XML paths failed"

    return stats
```
