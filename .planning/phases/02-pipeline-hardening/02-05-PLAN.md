**`get_freshness_report(session=None) -> dict`:**
- For each source in KNOWN_SOURCES:
  - Query: `SELECT max(completed_at) FROM pipeline_runs WHERE source=? AND status='success'`
  - If no row: `{"status": "never_run", "last_success": None, "hours_since": None}`
  - Else parse `completed_at` as ISO 8601, compute `hours_since = (now - completed_at).total_seconds() / 3600`
  - If `hours_since > STALE_THRESHOLD_HOURS`: `{"status": "stale", ...}`
  - Else: `{"status": "ok", ...}`
- Returns dict keyed by source name
- Accepts optional `session` param (if None, creates its own — supports both call patterns)

**`check_staleness(session=None) -> list[str]`:**
- Calls `get_freshness_report(session)`
- For each source with `status == "stale"`:
  - `logger.error("stale_source_detected", source=source, hours_since=X, threshold=STALE_THRESHOLD_HOURS)`
  - If `GRANTFLOW_ALERT_EMAIL` env var is set, send a plain-text email via `smtplib` to that address:
    ```
    Subject: [GrantFlow] Stale data alert: {source}
    Body: Source {source} has not successfully ingested in {hours_since:.1f} hours (threshold: 48h).
          Last success: {last_success}. Check pipeline logs.
    ```
    Use `SMTP_HOST` (default "localhost"), `SMTP_PORT` (default 25) env vars. If SMTP fails, log the error but do NOT raise — monitoring must never crash the pipeline.
- Returns list of stale source names (empty list = all OK)

**`normalize_cfda(raw: str | None) -> str`:**
- Input: any of "84.007", "84-007", "084.007", "84 007", "84.7", "  84.007  "
- Output: canonical "84.007" — two parts separated by dot: prefix (no leading zeros stripped) + suffix (3 digits zero-padded)
- Implementation:
  ```python
  def normalize_cfda(raw: str | None) -> str:
      if not raw:
          return ""
      raw = raw.strip()
      # Replace hyphens and spaces with dots
      raw = re.sub(r"[-\s]+", ".", raw)
      # Split on dot
      parts = raw.split(".")
      if len(parts) != 2:
          return raw  # can't normalize, return as-is
      prefix, suffix = parts
      # Remove leading zeros from prefix (84 not 084)
      prefix = str(int(prefix)) if prefix.isdigit() else prefix
      # Zero-pad suffix to 3 digits
      suffix = suffix.zfill(3) if suffix.isdigit() else suffix
      return f"{prefix}.{suffix}"
  ```

**`link_opportunities_to_awards(session=None) -> dict`:**
- Post-ingest pass that normalizes CFDA fields in-place and returns match stats
- Steps:
  1. Query all Opportunity records where `cfda_numbers IS NOT NULL AND cfda_numbers != ''`
  2. For each opportunity, normalize each CFDA number in the comma-separated list
  3. If the normalized value differs from stored value, update `opportunity.cfda_numbers` with normalized version
  4. Query Award records: for each opportunity, find awards where `normalize_cfda(award.cfda_numbers)` matches any of the opportunity's normalized CFDA numbers
  5. Log `opportunities_linked=N, awards_matched=M` via structlog
  6. Return `{"opportunities_processed": N, "cfda_normalized": K, "award_links_found": M}`

Implementation note on the ilike problem: Do NOT use `ilike` for the award link lookup — it's a full table scan (per CONCERNS.md). Instead, normalize CFDA numbers at ingest time (step 3 above) so they're stored in canonical format. Then the Award link query uses exact match: `Award.cfda_numbers.contains(normalized_cfda)`. This won't use an index perfectly but eliminates the format mismatch problem. A proper GIN index on cfda_numbers is a Phase 4 data quality concern.
