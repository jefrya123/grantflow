  <name>Task 2: POST /api/v1/keys endpoint and consistent error shape</name>
  <files>grantflow/api/keys.py, grantflow/app.py, tests/test_api_keys.py</files>
  <behavior>
    - POST /api/v1/keys accepts optional JSON body: {tier?: "free"|"starter"|"growth", label?: str}. Tier defaults to "free" if omitted.
    - Generates a secure random key: "gf_" + secrets.token_urlsafe(32) (total ~47 chars)
    - Stores SHA-256 hash of the full key string in api_keys.key_hash
    - Stores first 8 chars of key (after "gf_") in key_prefix for display
    - Returns {key: "<plaintext>", key_prefix: "gf_XXXXXX", tier: "free", created_at: "ISO8601"} — plaintext never stored, never returned again
    - Invalid tier returns 422 with {error_code: "INVALID_TIER", message: "tier must be one of: free, starter, growth"}
    - All error responses use shape: {"error_code": str, "message": str}. This shape applies to ALL api error responses going forward.
    - Test cases: successful creation returns key starting with "gf_", key_hash in DB is SHA-256 of returned key, invalid tier returns 422 with error_code field, key_prefix matches first 8 chars after "gf_", two calls return different keys
  </behavior>
  <action>
    Create grantflow/api/keys.py with a new APIRouter(prefix="/api/v1"). Import secrets, hashlib, datetime. The endpoint:
    1. Validates tier (reject unknown values with HTTPException 422 using error_code shape)
    2. Generates key = "gf_" + secrets.token_urlsafe(32)
    3. key_hash = hashlib.sha256(key.encode()).hexdigest()
    4. key_prefix = key[:8]  (covers "gf_" + 5 chars)
    5. Saves ApiKey row, commits, returns plaintext key in response

    Define a shared error response helper or use HTTPException detail as a dict: raise HTTPException(status_code=422, detail={"error_code": "INVALID_TIER", "message": "..."})
