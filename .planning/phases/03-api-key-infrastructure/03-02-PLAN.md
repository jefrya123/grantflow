    - Look up by key_hash in api_keys table where is_active=True
    - If not found: raise HTTPException(401, detail={"error_code": "INVALID_API_KEY", "message": "API key not found or inactive."})
    - Update last_used_at and increment request_count, commit
    - Return the ApiKey row
    - Tests: missing header → 401 MISSING_API_KEY, invalid key → 401 INVALID_API_KEY, valid key → returns ApiKey object with correct tier
