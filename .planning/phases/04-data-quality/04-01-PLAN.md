<objective>
Create a shared normalization layer (`grantflow/normalizers.py`) and wire it into all three ingest
modules so that all data entering the database is clean before storage.

Purpose: Downstream consumers (API, web UI, deduplication) operate on consistent, human-readable
values rather than raw government codes. This is the prerequisite for reliable deduplication (Plan 02)
and for the API to return useful eligibility filters.

- `grantflow/normalizers.py` — pure Python utility module, no DB dependencies
- Updated `grantflow/ingest/grants_gov.py`, `sbir.py`, `usaspending.py`

  <name>Task 1: Create grantflow/normalizers.py with all normalization functions</name>
  <files>grantflow/normalizers.py, tests/test_normalizers.py</files>
  <behavior>
    - normalize_date(value: str | None) -> str | None
      - Returns None for None or empty string
      - "01/15/2024" -> "2024-01-15" (MM/DD/YYYY)
      - "01152024" -> "2024-01-15" (MMDDYYYY)
      - "2024-01-15" -> "2024-01-15" (already ISO, passthrough)
      - "garbage" -> None (unparseable returns None, not raw value)
      - Supports formats: %m/%d/%Y, %m%d%Y, %Y-%m-%d, %Y%m%d, %b %d %Y
    - validate_award_amounts(floor: float | None, ceiling: float | None) -> tuple[float | None, float | None]
      - Returns (None, None) if either is negative
      - Returns (None, ceiling) if floor > ceiling (floor is invalid)
      - Returns (floor, ceiling) if floor <= ceiling and both non-negative
      - Returns (floor, None) / (None, ceiling) for single-sided values
    - normalize_eligibility_codes(codes: str | list | None) -> str
      - Input is a JSON string like '["12", "25"]' or a list or a bare string
      - Maps each code to human-readable label via ELIGIBILITY_CODE_MAP dict
      - Unknown codes → kept as-is (do not drop unknown codes)
      - Returns JSON string of translated labels: '["Nonprofits", "Small Businesses"]'
      - Returns '[]' for empty/None input
    - normalize_agency_name(name: str | None) -> str | None
      - Applies AGENCY_NAME_MAP dict for known inconsistencies across sources
      - Strips leading/trailing whitespace
      - Collapses multiple internal spaces to single space
      - Returns None for None/empty input

    ELIGIBILITY_CODE_MAP contents (Grants.gov applicant type codes):
      "00" -> "State governments"
      "01" -> "County governments"
      "02" -> "City or township governments"
      "04" -> "Special district governments"
      "05" -> "Independent school districts"
      "06" -> "Public and State controlled institutions of higher education"
      "07" -> "Native American tribal governments (Federally recognized)"
      "08" -> "Public housing authorities/Indian housing authorities"
      "11" -> "Native American tribal organizations (other than Federally recognized)"
      "12" -> "Nonprofits (other than higher education) with 501(c)(3) status"
      "13" -> "Nonprofits without 501(c)(3) status"
      "20" -> "Private institutions of higher education"
      "21" -> "Individuals"
      "22" -> "For-profit organizations (other than small businesses)"
      "23" -> "Small businesses"
      "25" -> "Others (see agency eligibility text)"
      "99" -> "Unrestricted"

    AGENCY_NAME_MAP: a dict of known variant forms to canonical forms, at minimum:
      "HHS" -> "Department of Health and Human Services"
      "Dept. of Health and Human Services" -> "Department of Health and Human Services"
      "Dept of Health and Human Services" -> "Department of Health and Human Services"
      "DOE" -> "Department of Energy"
      "Dept. of Energy" -> "Department of Energy"
      "Dept of Energy" -> "Department of Energy"
      "NSF" -> "National Science Foundation"
      "NIH" -> "National Institutes of Health"
      "USDA" -> "Department of Agriculture"
      (Add 10-15 most common federal agency abbreviations)
  </behavior>
