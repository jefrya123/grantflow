  <name>Task 1: Create grantflow/dedup.py with canonical ID logic</name>
  <files>grantflow/dedup.py, tests/test_dedup.py</files>
  <behavior>
    - make_canonical_id(opportunity: dict) -> str
      - Primary key: normalize and hash opportunity_number if present and non-empty
        - Normalize opportunity_number: strip whitespace, uppercase, collapse hyphens/spaces
        - e.g. "DE-FOA-0002345 " and "DE-FOA-0002345" produce the same canonical_id
      - Fallback: if no opportunity_number, hash (cfda_numbers + agency_code + close_date)
        - All fields normalized to lowercase, stripped, None becomes ""
      - Returns a deterministic hex string: "canon_" + sha256(normalized_key)[:16]
      - Two records with same normalized opportunity_number produce identical canonical_id
      - Two records with no opportunity_number but same cfda+agency+close_date produce identical canonical_id

    - find_duplicate_groups(session) -> list[dict]
      - Query: SELECT canonical_id, count(*), array_agg(id), array_agg(source)
        FROM opportunities GROUP BY canonical_id HAVING count(*) > 1
      - Returns list of dicts: {"canonical_id": str, "count": int, "ids": list[str], "sources": list[str]}
      - Read-only: does NOT modify any data
      - Empty list if no duplicates found

    - assign_canonical_ids(session) -> dict
      - For every opportunity in the database where canonical_id IS NULL:
        - Call make_canonical_id with a dict of the opportunity's fields
        - Set opportunity.canonical_id = result
      - Commits in batches of 1000
      - Returns stats: {"assigned": int, "already_set": int}
  </behavior>
  <action>

    - make_canonical_id is deterministic (same input always same output)
    - find_duplicate_groups returns empty list for session with no dupes (use a mock or in-memory SQLite)
