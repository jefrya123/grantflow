    1. In grantflow/normalizers.py, update normalize_date() to add a datetime.fromisoformat() pre-pass BEFORE the _DATE_FORMATS loop. This handles ISO 8601 with timezone offsets that strptime cannot parse. Implementation per research:
       ```python
       def normalize_date(value: str | None) -> str | None:
           if not value:
               return None
           value = value.strip()
           if not value:
               return None
           # Pre-pass: handle ISO 8601 with timezone offset (e.g. "2024-03-15T00:00:00-04:00")
           # datetime.fromisoformat() handles these natively in Python 3.11+
           try:
               dt = datetime.fromisoformat(value)
               return dt.strftime("%Y-%m-%d")
           except ValueError:
               pass
           for fmt in _DATE_FORMATS:
               try:
                   return datetime.strptime(value, fmt).strftime("%Y-%m-%d")
               except ValueError:
                   continue
           return None
       ```
       Note: fromisoformat() will also handle plain "YYYY-MM-DD" and "YYYY-MM-DDTHH:MM:SS" formats, so those will be caught earlier in the function. The _DATE_FORMATS loop still handles non-ISO formats (MM/DD/YYYY, YYYYMMDD, etc.). This is a performance improvement, not a behavior change, for those formats.
