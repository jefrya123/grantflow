    - test_tier_limit_starter: _tier_limit() returns "10000/day" for a starter-tier key hash
    - test_tier_limit_growth: _tier_limit() returns "100000/day" for a growth-tier key hash
    - test_tier_limit_unknown_key: _tier_limit() returns "1000/day" (free default) for unknown key string
    - test_tier_export_limit_free: _tier_export_limit() returns "100/day" for free-tier key
    - test_tier_export_limit_growth: _tier_export_limit() returns "10000/day" for growth-tier key
  </behavior>
  <action>
1. In `grantflow/api/auth.py`, add `_tier_limit(key: str) -> str` function below `TIER_LIMITS`:
   - Import `hashlib` (already imported) and `SessionLocal` from `grantflow.database`
   - Hash the incoming key with SHA-256
   - Open `SessionLocal()`, query `ApiKey` by `key_hash` and `is_active`, get tier (default "free")
   - Return `f"{TIER_LIMITS.get(tier, TIER_LIMITS['free'])}/day"`
   - Use `try/finally` with `db.close()` to prevent session leaks
   - CRITICAL: The `key` parameter name matters — slowapi inspects the callable signature and passes `key_func(request)` when parameter is named `key`

2. Add `_tier_export_limit(key: str) -> str` that returns `f"{TIER_LIMITS.get(tier, TIER_LIMITS['free']) // 10}/day"` (preserves 10:1 ratio: free=100, starter=1000, growth=10000).

3. In `grantflow/api/routes.py`:
   - Import `_tier_limit, _tier_export_limit` from `grantflow.api.auth`
   - Replace `@limiter.limit("1000/day")` with `@limiter.limit(_tier_limit)` on `search_opportunities`, `get_opportunity`, `get_stats`, `get_agencies` (4 routes)
   - Replace `@limiter.limit("100/day")` with `@limiter.limit(_tier_export_limit)` on `export_opportunities`
   - Do NOT change `/health` endpoint (no rate limit)

4. Write tests in `tests/test_auth_ratelimit.py` (file may have existing tests — add to it):
   - Create test fixtures that insert ApiKey rows with known tier values
   - Call `_tier_limit(raw_key)` directly and assert return string
   - Call `_tier_export_limit(raw_key)` directly and assert return string
   - Test unknown key returns free-tier default

ANTI-PATTERN: Do NOT try to use `request.state` — the limiter callback runs before FastAPI dependencies resolve.
ANTI-PATTERN: Do NOT use `asyncio.run()` in the callable — it runs in sync context within async event loop.
