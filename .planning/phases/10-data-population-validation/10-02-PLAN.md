    1. First, diagnose the SBIR crash. The ingestion_log shows status="running" with 0 records and no error -- meaning it crashed before reaching the except/finally block. Test each component in isolation:

       a. Test CSV download:
       ```
       uv run python -c "from grantflow.ingest.sbir import _download_csv; print(_download_csv())"
       ```
       This will either download the CSV or show the error. The CSV is ~100MB and may take several minutes. If it fails with a timeout, increase the httpx timeout or try a HEAD request first to verify the URL is valid.

       b. If download succeeds, check the CSV structure:
       ```
       uv run python -c "
       import csv
       with open('data/sbir_award_data.csv', 'r', errors='replace') as f:
           reader = csv.DictReader(f)
           print('Fields:', reader.fieldnames)
           for i, row in enumerate(reader):
               if i >= 3: break
               print(row)
       "
       ```
       Check if field names match CSV_FIELD_MAP keys in sbir.py (especially "award_year", "proposal_award_date", "firm", "award_title").

       c. If the CSV URL is dead or returns errors, check if sbir.gov changed the download URL. Try: `curl -I https://data.www.sbir.gov/awarddatapublic/award_data.csv` to see the HTTP status. If 404 or redirect, update SBIR_AWARDS_CSV_URL in config.py.

    2. Common fixes needed:
       - If CSV download times out: the timeout is already 600s which should be enough. If the server is slow, add `limits=httpx.Limits(max_connections=1)` to the httpx.stream call.
       - If CSV field names changed: update CSV_FIELD_MAP and any references to row.get("award_year"), row.get("proposal_award_date"), etc.
       - If the CSV URL is dead: check https://www.sbir.gov/sbirsearch/award/all for alternative download links. Update SBIR_AWARDS_CSV_URL in config.py.
       - If the solicitations API fails: the retry logic already handles this (3 attempts with backoff). If it consistently fails, log a warning but don't fail the whole ingestion -- awards are the primary data.

    3. Fix whatever the root cause is. The most likely issue based on the research is either:
       - Network timeout on the large CSV download (fix: already has 600s timeout, may need retry)
       - CSV encoding issue (fix: errors='replace' is already set)
       - Unhandled exception in _ingest_awards before the try/except in ingest_sbir catches it (fix: add more specific error handling)

    4. Clear the stuck "running" ingestion_log entry:

       uv run python -c "
